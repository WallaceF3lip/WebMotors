# 🧠 Exercícios Complementares — Programação Orientada a Objetos

Este repositório contém uma atividade prática com foco nos principais conceitos de **Programação Orientada a Objetos (POO)**.

---

## 📌 Objetivo

Desenvolver um sistema de **livre escolha**, aplicando os conceitos estudados em aula, com o objetivo de consolidar o aprendizado na prática.

---

## 🚀 Requisitos da Atividade

O sistema desenvolvido deve obrigatoriamente conter:

- 📦 **Classes**
- 🔐 **Atributos com controle de visibilidade**
- ⚙️ **Métodos**
- 🔒 **Encapsulamento**
- 🧬 **Herança**
- 🔄 **Getters e Setters**
- ➕ **Sobrecarga de métodos (Overload)**
- ♻️ **Reescrita de métodos (Override)**

---

## 👤 Forma de Entrega

- A atividade deve ser realizada **individualmente**
- O projeto deve ser versionado e disponibilizado no **GitHub**

---

## 💡 Tema escolhido - Automóveis

Escolhi um tema sobre a venda de veiculos, onde tenho a:

- 🚗 Representação de **veículos** como classes (Carro, Moto, Caminhão)
- 🏢 Gerenciamento de uma **concessionária**
- 🔐 Uso de **encapsulamento** para proteger dados sensíveis (ex: preço, dados)
- 🧬 Aplicação de **herança** (ex: classe Veiculo como base para Carro e Moto)
- 🔄 Utilização de **getters e setters** para acesso controlado aos atributos
- ♻️ Uso de **reescrita de métodos (override)** para comportamentos específicos de cada tipo de veículo
---

## 📚 Observação

O foco principal não é a complexidade do sistema, mas sim a **aplicação correta dos conceitos de POO**.
